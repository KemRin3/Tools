# 汎用画像切り機 (Generic Image Cutter)

`image_splitter.py` は、AI生成画像、スタンプシート、パーティクル素材、スプライトシートなどを `rows × cols` で切り出し、必要に応じてカット線を手動調整し、各セルを指定サイズのPNGとして保存するGUIツールです。

## 起動方法

```bash
python image_splitter.py
```

依存ライブラリとして Pillow が必要です。

```bash
python -m pip install Pillow
```

## 基本思想

処理順は以下に統一しています。

1. 画像を読み込む
2. `rows × cols` で均等グリッドを生成
3. カット線編集モードONの場合は、ユーザーが調整したカット線を使用
4. 各セルを切り出す
5. セル個別余白除去・センタリングONの場合は、各セルごとに有効ピクセルを検出
6. 不要な余白を削除
7. `padding` を追加
8. `output width × output height` の透明キャンバスを作成
9. 縦横比を維持して `contain` で収める
10. キャンバス中央に配置
11. PNGで保存

古い「画像全体をリサイズしてから分割」「全体画像の余白自動調整」「cover/stretch」「top/bottom alignment」など、この処理順と矛盾するUIは表示していません。

## 基本的な使い方

1. **画像ファイルを選択** で入力画像を選びます。
2. `rows` と `cols` に分割数を入力します。
3. `output width` と `output height` に保存する1セルのサイズを入力します。
4. 必要なら **カット線編集モード** をONにして、プレビュー上の線を調整します。
5. 必要なら **セル個別余白除去・センタリング** をONにします。
6. **出力フォルダを選択** で保存先を選びます。
7. **分割実行** を押すと、`元ファイル名_01.png` のような連番PNGが保存されます。

出力順は左上から右方向へ進み、次の行へ移動します。

## カット線編集モード

**カット線編集モード** は、`rows × cols` から作成した初期グリッドをGUI上で微調整してから書き出すモードです。AI生成したスタンプシートのように、各コマの境界が均等グリッドから少しずれている場合に使用します。

### 操作方法

1. 画像を読み込み、`rows` と `cols` を設定します。
2. **カット線編集モード** をONにします。
3. 「入力 / カット線」プレビュー上に表示される黄色の内側カット線をドラッグします。
4. 外枠は固定され、内側の縦線・横線だけを移動できます。
5. 線は隣接線を追い越さず、`min_cell_size` より小さいセルにならない範囲で移動します。
6. **分割実行** を押すと、現在のカット線座標で切り出したセルがPNG保存されます。

ドラッグ中の線は水色で表示され、現在の線座標は「選択中の線」欄に表示されます。線の位置は画像座標ベースで管理されます。

### スナップ

- `スナップ` をONにすると、ドラッグ中の線が指定単位に吸着します。
- 選択できる単位は `1`, `8`, `16`, `32`, `64` pxです。
- 512pxスナップはありません。

### リセット

**均等割り付けに戻す** を押すと、現在の `rows × cols` に基づく完全均等グリッドに戻ります。

### JSONプリセット

**カット線JSON保存** で現在のカット線をJSONとして保存できます。
**カット線JSON読み込み** で保存したプリセットを復元できます。

保存される内容は以下です。

```json
{
  "image_width": 1024,
  "image_height": 1024,
  "rows": 4,
  "cols": 4,
  "vertical_lines": [0, 256, 512, 768, 1024],
  "horizontal_lines": [0, 256, 512, 768, 1024]
}
```

読み込み時に、JSON内の `image_width` / `image_height` / `rows` / `cols` が現在の設定と一致しない場合は、クラッシュせずメッセージとログで通知します。

## セル個別余白除去・センタリング

**セル個別余白除去・センタリング** をONにすると、切り出した各セルを個別に処理します。

### detection mode

- `alpha`: 透明度で有効ピクセルを判定します。
- `brightness`: 黒背景素材向けに、輝度が `threshold` 以上のピクセルを有効ピクセルとして判定します。
- `auto`: 透明情報があれば `alpha`、なければ `brightness` を使用します。

### その他の設定

- `threshold`: brightness 判定で有効ピクセルとする輝度の下限です。
- `padding`: 検出した外接矩形の周囲に追加する透明余白です。
- `min_area`: 有効ピクセル数がこの値未満の場合、小さい孤立点として無視します。

有効ピクセルが検出できないセルは、ログに警告を出し、透明な空セルとして指定サイズに保存します。

## 出力仕様

- 出力形式はPNGです。
- ファイル名は `元ファイル名_01.png`, `元ファイル名_02.png` のような既存ルールを維持します。
- 出力順は左上から右方向、次の行へ進む順番です。
- 透過PNGは透過を保持します。
- すべての出力画像は必ず `output width × output height` になります。
- リサイズは縦横比を維持した `contain` のみです。
- 配置は中央配置のみです。
- 出力キャンバス背景は透明です。

## 動作確認方法

以下を確認してください。

1. `2×3`, `2×4`, `4×4` でグリッドが正しく出る。
2. カット線編集モードOFFの均等カットだけでPNG保存できる。
3. カット線編集モードONで線をドラッグして移動できる。
4. 線が外枠を超えない。
5. 線が隣の線を追い越さない。
6. `min_cell_size` が効く。
7. snap ON/OFF と `snap size` が効く。
8. **均等割り付けに戻す** で完全均等グリッドに戻る。
9. カット線JSON保存/読み込みができる。
10. 編集したカット線位置で正しくPNG分割される。
11. セル個別余白除去ON時に、各セルが `output width × output height` へ中央配置される。
12. 画像未選択、出力フォルダ未選択、rows/cols不正、output size不正、min_cell_size不正、JSON不一致などがクラッシュせず表示される。
