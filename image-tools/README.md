# Image Splitter

`image_splitter.py` は、1枚の画像を `rows × cols` のグリッドに分割して PNG として保存するデスクトップGUIツールです。

## 起動方法

```bash
python image_splitter.py
```

依存ライブラリとして Pillow が必要です。

```bash
python -m pip install Pillow
```

## 基本的な使い方

1. **画像ファイルを選択** で入力画像を選びます。
2. `rows` と `cols` に分割数を入力します。
3. 必要に応じて各モードの設定を変更します。
4. **出力フォルダを選択** で保存先を選びます。
5. **分割実行** を押すと、`元ファイル名_01.png` のような連番PNGが保存されます。

## 従来モード: 画像全体を処理してから分割

**指定サイズにリサイズしてから分割** をONにすると、画像全体を `width × height` にリサイズしてから分割します。

**自動余白調整** をONにすると、分割前に画像全体の不要な余白を検出して、中央基準で均等なキャンバスへ整えます。

- `判定モード`
  - `auto`: 透明ピクセルがあれば alpha、なければ brightness を使用します。
  - `alpha`: 透明部分を余白として扱います。
  - `brightness`: 黒背景素材向けに、輝度が `threshold` 以上のピクセルを有効領域として扱います。
- `threshold`: brightness 判定のしきい値です。初期値は `15` です。
- `padding`: 検出した外接矩形の周囲に追加する余白です。
- `正方形化`: 分割前のキャンバスを正方形にします。
- `中央寄せ`: 外接矩形の中心を基準に切り出します。
- `分割グリッド表示`: プレビュー上に分割グリッドを表示します。

このモードでは最終サイズが `rows` と `cols` で割り切れるよう、中央基準で端数を均等にカットまたは必要最小限で調整します。

## 分割後リサイズモード

**分割後リサイズモード** は、画像全体を先にリサイズするのではなく、元画像をそのまま `rows × cols` で仮分割してから各セルを個別に出力サイズへ整えるモードです。

処理順は以下です。

1. 元画像を `rows × cols` で仮分割します。
2. セル個別センタリングがONなら、有効ピクセルを検出して外接矩形を切り出します。
3. `padding` を追加します。
4. `fit mode` に従って `output cell width × output cell height` に収めます。
5. `background` 設定に従ってキャンバスを作成します。
6. `alignment` に従って配置します。
7. PNGで保存します。

### 設定

- `output cell width`: 保存する1セルの幅です。
- `output cell height`: 保存する1セルの高さです。
- `fit mode`
  - `contain`: 縦横比を維持し、素材全体がキャンバスに収まるようにリサイズします。
  - `cover`: 縦横比を維持し、キャンバス全体を埋めるようにリサイズします。はみ出した部分は中央基準でカットされます。
  - `stretch`: 縦横比を無視して、指定セルサイズへ強制リサイズします。
- `alignment`
  - `center`: 垂直方向も中央に配置します。
  - `top`: 上寄せで配置します。
  - `bottom`: 下寄せで配置します。
- `background`
  - `transparent`: 透明背景のキャンバスを作成します。
  - `black`: 黒背景のキャンバスを作成します。

## セル個別センタリングモード

**セル個別センタリング ON** は、仮分割した各セル内で有効ピクセルを検出し、セルごとに素材を中央へ配置するモードです。

粒子、魔法、光、煙など、各コマ内で素材位置や余白がばらつくスプライトシートにおすすめです。

### 設定

- `detection`
  - `auto`: 透明ピクセルがあれば alpha、なければ brightness を使用します。
  - `alpha`: 透明度で有効ピクセルを判定します。
  - `brightness`: 黒背景素材向けに、輝度で有効ピクセルを判定します。
- `background`
  - `transparent`: 切り出し・最終キャンバスの背景を透明にします。
  - `black`: 切り出し・最終キャンバスの背景を黒にします。
- `threshold`: brightness 判定で有効ピクセルとする輝度の下限です。
- `padding`: 検出した外接矩形の周囲に追加する余白です。
- `scale limit`: セル個別センタリング後に `contain` / `cover` で拡大しすぎないよう、倍率の上限を指定します。
- `min_area`: 有効ピクセル数がこの値未満の場合は孤立点として無視します。

## モードの併用

**分割後リサイズモード** と **セル個別センタリング ON** は併用できます。

併用時は、まず元画像を仮分割し、各セルで有効ピクセル検出・切り出し・padding追加を行ってから、指定した `output cell width × output cell height` にリサイズし、指定背景のキャンバスへ配置して保存します。

> 注意: 分割後リサイズモードまたはセル個別センタリングモードを使う場合、従来の「画像全体をリサイズしてから分割」および「画像全体の自動余白調整」は適用されません。元画像を先に仮分割してからセルごとに処理します。

## カット線編集モード

**カット線編集モード** は、`rows × cols` から作成した初期グリッドをGUI上で微調整してから書き出すモードです。AI生成したスタンプシートや素材シートのように、各コマの境界が均等グリッドから少しずれている場合に使用します。

### 操作方法

1. 画像を読み込み、`rows` と `cols` を設定します。
2. **カット線編集モード** をONにします。
3. 「処理前 / カット線編集」プレビュー上に表示される黄色の内側カット線をドラッグします。
4. 外枠は固定され、内側の縦線・横線だけを移動できます。
5. 線は隣接線を追い越さず、`min_cell_size` より小さいセルにならない範囲で移動します。
6. **分割実行** を押すと、現在のカット線座標で切り出したセルがPNG保存されます。

ドラッグ中の線は水色で表示され、現在の線座標は「選択中の線」欄に表示されます。

### スナップ

- `スナップ` をONにすると、ドラッグ中の線が指定単位に吸着します。
- 選択できる単位は `1`, `8`, `16`, `32`, `64` pxです。
- 512pxスナップはありません。

### リセット

**均等割り付けに戻す** を押すと、現在の `rows × cols` に基づく完全均等グリッドに戻ります。

### JSONプリセット

**カット線JSON保存** で現在のカット線をJSONとして保存できます。
**カット線JSON読み込み** で保存したプリセットを復元できます。

保存される内容は以下です。

```json
{
  "image_width": 1024,
  "image_height": 1024,
  "rows": 4,
  "cols": 4,
  "vertical_lines": [0, 256, 512, 768, 1024],
  "horizontal_lines": [0, 256, 512, 768, 1024]
}
```

読み込み時に、JSON内の画像サイズ・`rows`・`cols` が現在の設定と一致しない場合はエラーになります。

### 書き出し時の動作

カット線編集モードがOFFの場合は、従来の通常分割処理を使用します。

カット線編集モードがONの場合は、固定512分割や均等分割ではなく、現在の `vertical_lines` / `horizontal_lines` を使って元画像から各セルを切り出します。切り出したセルは `output cell width × output cell height` のキャンバスへ出力され、`fit mode`、`alignment`、`background` の既存設定に従って配置されます。

**セル個別センタリング ON** と併用した場合は、カット線で切り出した後に、各セル内で Alpha / Brightness / Auto 判定による有効ピクセル検出、bbox切り出し、padding追加、scale limit、min_area の処理を適用してから保存します。
