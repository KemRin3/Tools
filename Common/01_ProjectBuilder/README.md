# 01_ProjectBuilder

`01_ProjectBuilder` は、DSL形式のテキストからフォルダ、空ファイル、内容付きファイルを生成する Python/tkinter ツールです。

## 特徴

- `dir:` でフォルダを生成します。
- `file:` で空ファイルを生成します。
- `write:` で内容付きファイルを生成します。
- `---` で `write:` ブロックを終了します。
- UTF-8 でファイルを書き込みます。
- Windows での利用を想定しています。
- GUI で選択したフォルダを `base_dir` とし、DSL内のパスはその配下の相対パスとして扱います。
- `presets/` 内の `.txt` プリセットを UTF-8 で読み込み・保存できます。
- 指定フォルダ配下の構成を解析し、ProjectBuilder DSL に変換できます。
- 作成ログを GUI に表示します。
- 同名のフォルダまたはファイルが存在する場合はスキップします。

## 起動方法

```powershell
python Common\01_ProjectBuilder\main.py
```

## プリセット

`Common\01_ProjectBuilder\presets\` にある `.txt` ファイルをプリセットとして扱います。

- `Refresh`: `presets/` 内の `.txt` ファイル一覧を更新します。
- `Load`: 選択したプリセットを UTF-8 で読み込み、DSL 入力欄に反映します。
- `Save`: 現在の DSL 入力欄の内容を UTF-8 で `presets/` に保存します。

保存時に拡張子を省略した場合は `.txt` を自動で付けます。既存ファイルがある場合は上書き確認を表示します。

## 指定フォルダの解析

`Analyze folder` で解析対象フォルダを選択すると、そのフォルダの中身だけを DSL 化して DSL 入力欄へ出力します。解析対象フォルダ自身は出力しません。

出力ルール:

- フォルダは `dir:` で出力します。
- ファイルは `file:` で出力します。
- ファイル内容は読みません。
- パスは解析対象フォルダからの相対パスで出力します。
- パス区切りは `/` で出力します。
- 隠しファイル・隠しフォルダ、`__pycache__`、`logs/`、`.git/` は除外します。

例: `素材作成/` を解析した場合

```text
dir:00_rules/
file:00_rules/texture_rules.md
file:00_rules/prompt_rules.md
dir:01_source/
dir:01_source/particles/
```

## EXE化する方法

Windows では PyInstaller を使うのが簡単です。以下は PowerShell での手順です。

### 1. 仮想環境を作成する

```powershell
cd C:\path\to\Tools
py -3.12 -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
```

### 2. PyInstaller をインストールする

```powershell
python -m pip install pyinstaller
```

### 3. EXEを作成する

```powershell
python -m PyInstaller --onefile --windowed --name ProjectBuilder Common\01_ProjectBuilder\main.py
```

作成後、以下の EXE を実行します。

```powershell
.\dist\ProjectBuilder.exe
```

### オプション

- `--onefile`: 1つの EXE にまとめます。
- `--windowed`: コンソール画面を表示せず GUI のみ起動します。
- `--name ProjectBuilder`: 出力 EXE 名を `ProjectBuilder.exe` にします。

### 再ビルド時のクリーンアップ

ビルド結果を作り直したい場合は、`build`、`dist`、`ProjectBuilder.spec` を削除してから再実行します。

```powershell
Remove-Item -Recurse -Force build, dist
Remove-Item -Force ProjectBuilder.spec
python -m PyInstaller --onefile --windowed --name ProjectBuilder Common\01_ProjectBuilder\main.py
```

## DSL 仕様

GUI の「Select base folder」で作成先フォルダを選択してから実行します。たとえば `C:\work` を選択した場合、以下の DSL は `C:\work\素材作成\00_rules` 配下に作成されます。

```text
dir: 素材作成/00_rules/
file: 素材作成/00_rules/empty.txt
write: 素材作成/00_rules/readme.md
# 素材作成ルール

Generated by project_builder.
---
```

### コマンド

| コマンド | 内容 |
| --- | --- |
| `dir:` | `base_dir` 配下の指定した相対パスにフォルダを作成します。 |
| `file:` | `base_dir` 配下の指定した相対パスに空ファイルを作成します。 |
| `write:` | `base_dir` 配下の指定した相対パスに内容付きファイルを作成します。 |
| `---` | `write:` の内容入力を終了します。 |

## 注意事項

- すべてのパスは GUI で選択した `base_dir` からの相対パスで指定してください。
- `C:/` や `D:/` などのドライブから始まる絶対パスは禁止です。
- `/` から始まる絶対パスは禁止です。
- `../` や `..\\` など、親ディレクトリへ移動する指定は禁止です。
- 既存の同名パスは上書きせず、ログに `SKIP exists` と表示します。
- 作成ログには実際の作成先フルパスを表示します。
- `file:` と `write:` の親フォルダが存在しない場合は自動作成します。
