# project_builder

`project_builder` は、DSL形式のテキストからフォルダ、空ファイル、内容付きファイルを生成する Python/tkinter ツールです。

## 特徴

- `dir:` でフォルダを生成します。
- `file:` で空ファイルを生成します。
- `write:` で内容付きファイルを生成します。
- `---` で `write:` ブロックを終了します。
- UTF-8 でファイルを書き込みます。
- Windows での利用を想定しています。
- GUI で選択したフォルダを `base_dir` とし、DSL内のパスはその配下の相対パスとして扱います。
- 作成ログを GUI に表示します。
- 同名のフォルダまたはファイルが存在する場合はスキップします。

## 起動方法

```powershell
python Common\project_builder\main.py
```

## EXE化する方法

Windows では PyInstaller を使うのが簡単です。以下は PowerShell での手順です。

### 1. 仮想環境を作成する

```powershell
cd C:\path\to\Tools
py -3.12 -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
```

### 2. PyInstaller をインストールする

```powershell
python -m pip install pyinstaller
```

### 3. EXEを作成する

```powershell
python -m PyInstaller --onefile --windowed --name ProjectBuilder Common\project_builder\main.py
```

作成後、以下の EXE を実行します。

```powershell
.\dist\ProjectBuilder.exe
```

### オプション

- `--onefile`: 1つの EXE にまとめます。
- `--windowed`: コンソール画面を表示せず GUI のみ起動します。
- `--name ProjectBuilder`: 出力 EXE 名を `ProjectBuilder.exe` にします。

### 再ビルド時のクリーンアップ

ビルド結果を作り直したい場合は、`build`、`dist`、`ProjectBuilder.spec` を削除してから再実行します。

```powershell
Remove-Item -Recurse -Force build, dist
Remove-Item -Force ProjectBuilder.spec
python -m PyInstaller --onefile --windowed --name ProjectBuilder Common\project_builder\main.py
```

## DSL 仕様

GUI の「Select base folder」で作成先フォルダを選択してから実行します。たとえば `C:\work` を選択した場合、以下の DSL は `C:\work\素材作成\00_rules` 配下に作成されます。

```text
dir: 素材作成/00_rules/
file: 素材作成/00_rules/empty.txt
write: 素材作成/00_rules/readme.md
# 素材作成ルール

Generated by project_builder.
---
```

### コマンド

| コマンド | 内容 |
| --- | --- |
| `dir:` | `base_dir` 配下の指定した相対パスにフォルダを作成します。 |
| `file:` | `base_dir` 配下の指定した相対パスに空ファイルを作成します。 |
| `write:` | `base_dir` 配下の指定した相対パスに内容付きファイルを作成します。 |
| `---` | `write:` の内容入力を終了します。 |

## 注意事項

- すべてのパスは GUI で選択した `base_dir` からの相対パスで指定してください。
- `C:/` や `D:/` などのドライブから始まる絶対パスは禁止です。
- `/` から始まる絶対パスは禁止です。
- `../` や `..\\` など、親ディレクトリへ移動する指定は禁止です。
- 既存の同名パスは上書きせず、ログに `SKIP exists` と表示します。
- 作成ログには実際の作成先フルパスを表示します。
- `file:` と `write:` の親フォルダが存在しない場合は自動作成します。
